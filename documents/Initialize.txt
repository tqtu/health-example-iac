🚀 Infrastructure Setup Guide (Public)
1. Prerequisites

    AWS CLI installed and configured (aws configure).

    Terraform (v1.7.0+) installed locally.

    GitHub Repository with "Environments" enabled.

2. Initialize the Remote Backend

Before running the pipeline, you must create a "Storage" area in your AWS account so Terraform can track your resources securely. Run these commands in your terminal:
Bash

# 1. Create a unique S3 bucket for Terraform State
aws s3 mb s3://YOUR-UNIQUE-STATE-BUCKET-NAME --region ap-southeast-2

# 2. Enable Versioning (Safety feature)
aws s3api put-bucket-versioning --bucket YOUR-UNIQUE-STATE-BUCKET-NAME --versioning-configuration Status=Enabled

# 3. Create a DynamoDB table for State Locking
aws dynamodb create-table \
    --table-name terraform-state-locking \
    --attribute-definitions AttributeName=LockID,AttributeType=S \
    --key-schema AttributeName=LockID,KeyType=HASH \
    --provisioned-throughput ReadCapacityUnits=5,WriteCapacityUnits=5 \
    --region ap-southeast-2
